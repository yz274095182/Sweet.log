# 📱 产品需求文档 (PRD)：专属控糖手记

## 1. 产品概述

- **产品愿景：** 打造一款有温度、无负担的轻量级血糖管理工具，将冰冷的医疗数据转化为温馨的生活记录与情绪抚慰。
- **目标用户：** 核心为妊娠期孕妇（有严格控糖需求、需高频记录且易产生焦虑情绪），同时向下兼容常规糖尿病患者或控糖减脂人群。
- **核心亮点：**
  - **极简交互：** 智能时段识别 + 滑动刻度尺，实现单手秒级记录。
  - **情绪抚慰：** 告别冰冷的“提交成功”，提供基于数据状态的随机温情文案。
  - **数据快照与导出：** 灵活的医嘱阈值配置，一键生成报表，方便复诊。
  - **零门槛：** 基于微信云开发，免注册登录，打开即用。

## 2. 核心业务流程 (User Flow)

### 2.1 首次启动初始化 (Onboarding)

1. 用户首次授权打开小程序。
2. 弹出全屏或居中轻量级卡片，文案引导：“为了提供最适合您的健康标准，请问您目前处于孕期吗？”
3. 提供两个大按钮：【🤰 是的，我在孕期】/【🧑‍🤝‍🧑 否，常规控糖】。
4. 后台静默创建 `users` 表记录，绑定 `openid`。根据选择的类型，读取前端常量字典（GDM 或 NORMAL），生成初始的 `threshold_config`（阈值快照）写入数据库，完成初始化，进入首页。

### 2.2 日常打卡流程

打开小程序 ➡️ 首页自动高亮当前时段 ➡️ 核心大字展示该时段正常值中位数 ➡️ 用户滑动刻度/微调/直接输入 ➡️ （可选）填写饮食备注 ➡️ 点击保存 ➡️ 触发防重复校验（如有则覆盖） ➡️ 弹出情绪反馈 Emoji 与文案 ➡️ 自动切回首页或图表页。

## 3. 功能模块详细设计

### 3.1 极简记录模块 (首页)

- **智能时段标签 (Time Tag)：**
  - 根据当前设备系统时间，自动匹配并高亮 7 大标准时段之一（早餐前、早餐后、午餐前、午餐后、晚餐前、晚餐后、睡前）。
  - **交互：** 点击当前标签，底部平滑弹出面板（Bottom Sheet），展示所有 7 个时段的网格/列表，支持手动修正归类时段。
- **双时间维度与防重复机制：**
  - **底层记录：** 系统不仅记录归类的 `time_tag`，同时获取用户点击保存那一刻的**绝对时间戳**（或允许用户手动修改的精确时间）。
  - **同日同标签覆盖：** 若用户在同一天内，对同一标签（如今天 2026-04-12 的“早餐前”）进行二次提交，前端需弹窗拦截：“您今天已记录过 [早餐前] 的血糖（X.X），是否用新数据覆盖？”确认后，执行数据库 `update` 操作。
- **核心数值录入 (Glucose Input)：**
  - **默认值：** 读取数据库中该用户当前时段对应的“正常范围中位数”作为初始显示值。
  - **中心主干：** 特大号数字显示，点击该数字直接唤起系统数字小键盘输入。
  - **滑动刻度尺：** 数字下方配置横向滑动刻度尺，支持左右拖动大跨度选值。
  - **微调按钮：** 刻度尺两侧配备 `+0.1` 和 `-0.1` 按钮。
- **状态与备注 (Note)：**
  - 提供轻量级文本输入框，提示：“记录一下吃了什么，或者现在的感受吧～”。
- **提交与情绪反馈：**
  - 根据数值所处区间（达标/偏高/偏低），从文案矩阵中随机抽取一条弹窗提示。

### 3.2 数据可视化模块 (图表页)

- **多系列折线图 (Multi-line Chart)：**
  - **坐标轴：** X轴为日期时间（支持按最近 7 天 / 30 天切换），Y轴为血糖绝对值。
  - **数据展示：** 7 个时段标签对应 7 条独立折线。
  - **交互：** 顶部提供图例（Legend），支持点击显示/隐藏特定时段的数据线（例如只看“早餐后”的趋势）。
- **动态警戒带 (Threshold Area)：**
  - 图表背景中绘制两条水平警戒线（上限与下限），形成“安全区间带”。
  - 警戒带的取值动态读取用户 `users` 表中的 `threshold_config`。

### 3.3 个性化与设置模块 (我的)

- **基础信息：** 展示用户微信头像与昵称（基于微信最新头像昵称获取规则）。
- **医嘱阈值自定义：**
  - 允许用户进入“高级设置”，覆盖系统默认的高低警戒线。修改后更新数据库的 `threshold_config`。
  - 提示文案：“系统标准仅供参考，请优先遵从主治医生建议进行调整。”
- **数据导出 (第一期核心功能)：**
  - 提供“生成本月数据报表”按钮。
  - **技术实现：** 调用微信云函数，在云端查询该用户本月数据，利用 Node.js 库（如 `exceljs`）生成 Excel 文件存入云存储，前端获取临时链接后调用 `wx.downloadFile` 和 `wx.openDocument`，供用户预览并转发给微信好友/医生。

## 5. 数据库架构设计 (微信云开发)

### 5.1 `users` 集合 (用户表)

采用配置快照模式，用户实际生效的警戒线仅以此表中的 `threshold_config` 为准。

- `_id`: String (系统生成)
- `_openid`: String (微信用户唯一标识)
- `diabetes_type`: String (枚举: 'GDM' 或 'NORMAL')
- `threshold_config`: Object (核心配置项，保存当前用户各时段的高低阈值快照)
- `created_at`: Timestamp
- `updated_at`: Timestamp

### 5.2 `records` 集合 (记录表)

- `_id`: String (系统生成)
- `_openid`: String (关联用户)
- `date_str`: String (格式 "YYYY-MM-DD"，高频用于同日同标签覆盖查询及按月出表)
- `time_tag`: String (枚举: '早餐前', '早餐后' 等)
- `record_time`: Timestamp (精确到秒的真实测糖时间)
- `glucose_value`: Number (血糖数值)
- `note`: String (可选，备注内容)
- `created_at`: Timestamp (数据首次写入时间)
- `updated_at`: Timestamp (触发覆盖机制时的更新时间)
