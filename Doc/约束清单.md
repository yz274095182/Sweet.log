# 🤖 致 AI 编程助手的架构约束清单 (Uni-app x 纯血原生版)

**项目基调：** 本项目目标是基于 DCloud 最新一代跨端引擎打造极高性能的控糖记录应用。优先发布微信小程序，未来将通过 UTS 编译为纯原生 iOS/Android App。请严格遵循 `Uni-app x` 的特殊工程规范生成代码。

## 1. 核心技术栈与严格版本锁 (Bleeding-edge Stack)

- **底层环境：** **Node.js v24.x LTS**。
- **包管理器：** 强制使用 **pnpm**。需在项目根目录配置 `.npmrc` (`shamefully-hoist=true`)。
- **核心框架：** 强制使用 **Uni-app x**。
  - _文件规范：_ 所有页面和业务组件必须使用 **`.uvue`** 后缀（严禁使用 `.vue`）。
  - _语法范式：_ 必须使用 `<script setup lang="uts">` 组合式 API，严格遵守 UTS (Uni TypeScript) 强类型检查规范。
- **构建工具：** Vite。
- **CSS 方案 (极度高危)：** \* 强制使用 **Tailwind CSS v3.4.x**。
  - **样式底线：** 由于 Uni-app x 的原生 App 端仅支持 Flexbox 布局和有限的 CSS 属性，AI 生成的 Tailwind 类名 **必须仅限于基础的 Flex、Margin、Padding、Color、Border、Font 等跨端兼容的原子类**。严禁使用 `grid`, `float`, `box-shadow` (原生用 `elevation`) 等纯 Web 高级属性。

## 2. 状态管理与数据可视化

- **状态：** 强制使用 **Pinia**，采用纯 Composition API (Setup Stores) 风格。
- **图表：** 必须使用 **uCharts** (`qiun-data-charts` 组件化版本) 确保多端 Canvas 渲染的高性能与稳定性。绝对禁止引入 ECharts。

## 3. 工程规范与性能 (Directory & Performance)

- **目录结构：** 严格遵循 Uni-app x 的 `pages.json` 和 `easycom` 组件自动引入规范。
- **类型安全：** 所有变量、函数参数和返回值必须具备明确的 UTS 类型声明，杜绝使用 `any`。
- **性能防抖：** 核心高频交互组件（如首页滑动刻度尺）的 `touch` 事件与震动反馈 (`uni.vibrateShort`) 必须在逻辑层做好 `throttle/debounce` 性能优化。
- **职责分离：** 页面文件（`.uvue`）仅处理 UI 渲染。与微信云开发数据库的交互、覆盖判定等业务逻辑，必须抽离到 `services` 或 `api` 目录的 `.uts` 文件中。
